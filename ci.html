Compilers and interpreters

Compilers and interpreters are different methods at turning high level code into machine code.

Compilers

Takes the whole source code as one and converts it to machine code.

Advantages

Faster at running

Dis-advantages

Takes longer to compile at first, which is bad for developers

Interpreter

Takes one line at a time and runs it.

Advantages

Easier to track errors in the code to the line

Dis-advantages

It takes longer to run each line - making a slower program.

Translator

Takes the user's high level code and turns it into machine code.

the feeling of multiprocess.

Bytecode/Assemble

Computer object that is processed by a program. The code can run on any platform as it is a list of bytecode instructions.

Compilers stages

Stage 1 - Lexical analysiys

Prepares the code to be translated. It makes a list of all the characters in the high level code, in one lone line.

A symbol table is created to work out what every variable, symbol and number is and gives it a numeric value.

Stage 2 - Syntax analasys

This part determinds whether the code is correct or not. The Compiler will have it's own set of

rules. It will simply reference its current syntax and see if it works.

Stage 2 -Semantic analasys

Check to see if the meaning of the symbols in the code are correct. For example: asigning an string to an int.
