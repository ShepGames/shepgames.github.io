Common types of paradigms:

Event-Driven

A type of programming which relies on user input

to run specific actions when the user wants to, such as buttons that change parts of the program. Commonly

used in HTML websites that contain JavaScript functions.

Object-Oriented

A type of programming which sorts different functions and variables for those

functions into objects and attributes, which can then be used to change or modify parts of the object to fit

with the execution of the program. Used often in C++.

Procedural

A type of programming which is one of the simplest; all the code in this type runs sequentially,

one line after the other, but it can use functions to change what order the code will run in.

Often associated with Python, despite Python fitting

the previous 2 paradigm formats as well.

Imperative

A type of programming that is heavily focused on statements to shift the state

of the program, and utilizes direct commands to do so. Object-Oriented programming was developed from

this type of paradigm, and it

is also one of the oldest paradigms of all time, as it was used in FORTRAN, developed by IBM in 1954.

Declarative

A type of programming that does not necessarily involve heavy amounts of complex language but instead

uses high amounts of logic, as the code is more focused on defining the expected output for the program than

the way it should be carried out. Used often with SQL.
