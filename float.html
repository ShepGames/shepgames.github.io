Floating point

Normal decimal numbers can be represented with standard form. So can binary numbers

We can represent Binary using the 8 4 2 1 base 2, and can do the other side of the decimal point. 8 2 4 1.0.5 0.25 0.125 and so on

Some numbers are too big or too small to fit in a normal 8-bit number, this is where we use a mantissa and an exponent.

The Mantissa is the Binary number you are given.

The exponent is the number of places you are moving to. It is all in base two so the exponent 2 to the power of the number given.

010010 0110 is the typical way of writing this.

In every question like this we add a decimal point in-between the first digit and the second digit of the mantissa.

Be aware that if this is in a sign and magnitude you will need to have the first character of both the exponent and mantissa as a negative number

Positive exponents move the decimal right, and negative to the left

Example:

1)Normalize 1001101 0101

Firstly we convert the exponent to denary, in order to know how many places to move by. This is 5

Because the data is signed we must stay aware if the number was 1101 it would be -5

Add in the decimal point, then slide it 5 places right: 1.001101 --> 100110.1

Our normalized number is: 100110.1 and in denery -6.5 as it is signed
