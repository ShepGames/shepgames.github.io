Computer Science A level

Binary

Binary is simply the way to represent numbers in a computer system. 1s and 0s represent on and off.

Binary uses base-two to store numbers.This means the far right digit represtnes 1, the next one to the left is 2, the next 4...

A single number such as 1 or 0 is known as a bit

4 bits is a nibble

8 bits is a byte

We typically use bytes to represent ASCII, Unicode uses 16-bits.

Example:

1)Convert the number 115 to binary

Firstly we write out the base-two numbers, and go from the left across working out what to add

128,64,32,16,8,4,2,1

0 , 1, 1, 1,0,0,1,1

We see 128 is too big, so put a 0. 64 is in the number so we put that in, we further add 32 to get it higher.

When we get to 8, we see that will be too big so do not put it in.

This gives us an answer of 01110011

Hex

Hex is a way we represent binary numbers and characters in an ASCII chart.

We have to remember that the value 10 is equal to A in the hex world. From there we can count

up to 15 which is F. When converting to hex we need an 8-bit binary number. If you are given a denary number,

you will need to convert it to binary. With the binary number, you will split it in half to 4-bit nibbles.

Calculate the value of each 4 bit number, the highest it can be is 15.

Write the value you get under neath each number, and find its corrosponding letter if needed. This is your hex value.

Example:

1)Convert the binary number 10010110 to Hex

Firstly we split it to nibbles

1001 0110

we then convert to denary

(8+1) (4+2) = 10 and 6

10 is A so our hex value is A6

Adding Binary numbers

You will be likely asked to add binary numbers. To do this we must remember some rules:

0+0=0

0+1=1

1+1=10, or 0 and carry the 1

1+1+1=11 or 1 carry the 1

If you get a number bigger than 8-bits, we call this an overflow error. It means the number is too big to be stored.

This can cause potential problems as the number may not be represented fully in the system.

Example:

1)Add the binary numbers 10010110 to 01101110

Firstly we line them up, then start on the right and apply our rules

10010110

01101110+

--------

100000100

1111111

Here we have kepts carrying the 1 and have an overflow error.

Our answer is: 100000100
